# 本地 Excel 赔率文件解析指南（2026-06-13更新）

当外部爬虫（500.com/titan007）无法获取早期比赛赔率时，用户提供的本地 `.xls`/`.xlsx` 文件是主要补充源。

## 文件位置

用户通常将赔率 Excel 文件保存在 `Downloads` 目录下。文件名格式示例：
- `加拿大VS波黑(世界杯)欧洲数据.xls`
- `加拿大VS波黑(亚盘).xls`
- `加拿大VS波黑(大小).xls`
- `加拿大VS波黑(让球指数).xls`

每场比赛涉及多个 Excel 文件（欧赔 + 亚盘 + 大小球 + 让球指数），文件名含比赛名和品类关键词。

## 详细列布局

### 1. 欧赔（欧洲数据）

**表头格式**（两行表头）：
```
Row 0: ['欧赔公司', '欧赔', '', '', '', '', '', '', '凯利指数', '', '']
Row 1: ['', ' 胜 ', ' 平 ', ' 负 ', '胜率', '和率', '负率', '返还率', '', '', '']
```

**数据行格式**（每家占两行，一即时一初盘）：

| 列 | 内容 | 备注 |
|:-|:-|:-|
| col0 | 公司名称 | 混名格式（见下方） |
| col1 | 主胜赔率 | float |
| col2 | 平局赔率 | float |
| col3 | 客胜赔率 | float |
| col4 | 胜率 = (1/col1) × 返还率 | float, 含抽水 |
| col5 | 和率 = (1/col2) × 返还率 | float |
| col6 | 负率 = (1/col3) × 返还率 | float |
| col7 | 返还率 | float, ~0.88-0.95 |
| col8 | 凯利(胜) = col1 × 胜率 | float |
| col9 | 凯利(平) = col2 × 和率 | float |
| col10 | 凯利(负) = col3 × 负率 | float |

**示例**（威廉希尔即时盘）：
```python
['威廉希尔', 2.0, 3.5, 3.25, 0.4573, 0.2613, 0.2814, 0.9146, 1.0393, 0.9592, 0.6704]
```

**关键**：凯利指数的值 = 公司赔率 × 市场平均胜率。凯利 > 1.0 表示该公司赔率高于"公平赔率"（正EV信号），凯利 < 0.95 表示公司谨慎控赔。

## 两行结构陷阱

欧赔表中**每家公司占两行**：第一行为即时赔率+凯利，第二行为初始赔率+凯利。如果解析"按行遍历→每行当作一家公司"，会得到两倍结果：

```python
# Row 12: ['威廉希尔', 2.0, 3.5, 3.25, ...]      ← 即时
# Row 13: ['', 1.8, 3.3, 4.5, ...]               ← 初始（空名）
# Row 14: ['澳门', 1.82, 3.13, 4.05, ...]         ← 即时
# Row 15: ['', 1.62, 3.15, 4.75, ...]             ← 初始
```

**正确的解析逻辑**：
1. 只取 col0 非空且有值的行 → 这样只取每家"即时盘"（第1行）
2. 或者取两行分别标注 `type: 'current'` 和 `type: 'initial'`

**如果解析后公司数翻倍（如55→110），说明没过滤空名行。**

## EV 计算方法

从本地欧赔 Excel 计算真实期望值的步骤：

### Step 1: 市场共识概率

用所有公司（排除最高/最低/平均值行）的赔率计算平均去水概率：

```python
aw = sum(c['win'] for c in companies) / len(companies)
ad = sum(c['draw'] for c in companies) / len(companies)
al = sum(c['lose'] for c in companies) / len(companies)

# 去水
tot = 1/aw + 1/ad + 1/al
prob_w = (1/aw) / tot
prob_d = (1/ad) / tot
prob_l = (1/al) / tot

# 公平赔率
fair_w = 1/prob_w
fair_d = 1/prob_d
fair_l = 1/prob_l
```

### Step 2: 逐公司 EV

```python
for company in companies:
    ev_w = (company['win'] / fair_w - 1) * 100   # %
    ev_d = (company['draw'] / fair_d - 1) * 100
    ev_l = (company['lose'] / fair_l - 1) * 100

    if ev_w > 0:
        print(f"正EV: 主胜 {company['name']} @ {company['win']:.2f} EV={ev_w:.2f}%")
```

### Step 3: 凯利交叉验证

凯利指数已预计算在 Excel 中（col8-10）。三种情况：
- **凯利 > 1.0** → 公司赔率高于公平价 → 确认正EV
- **凯利 < 0.95** → 公司谨慎控赔 → 市场可能不看好该结果
- **凯利 ≈ 0.97-1.02** → 正常范围

### Step 4: 亚盘方向信号

- 升盘（如 `半球 升`）→ 资金涌入让球方 → 支持让球方
- 降盘（如 `半球 降`）→ 资金流出 → 支持下盘
- 上盘水位 < 0.80 → 上盘受热明显

## 综合推荐逻辑

1. 从正EV列表取 EV 最高的选项
2. 验证凯利 > 1.0 或接近 1.0（非 < 0.95）
3. 检查亚盘方向是否一致
4. 低概率方向（< 20%）标注为"高风险冷门"，即使 EV 高也需降注
5. 输出表格格式

## 端到端工作流：从Excel到投注单

按以下步骤从零生成可执行投注方案：

### Step 0: 合并多Excel文件 → all_matches_odds_data.json

每个比赛日通常有多个 `.xls` 文件（欧赔/亚盘/大小球/让球各一个），先用 `xlrd` 读取后合并为统一结构：

```json
{
  "加拿大VS波黑": {
    "match": "加拿大VS波黑",
    "oude": [{"sheet": "...", "filename": "...", "rows": [...]}],
    "yapan": [...],
    "daxiao": [...],
    "rangqiu": [...]
  }
}
```

### Step 1: 解析 rows → 结构化公司数据

从合 JSON 的每个比赛字段中取出 `[0]['rows']`，传递给解析函数。

**关键经验**（2026-06-13 迭代验证）：
- 欧赔时**每家公司只占1行**（col0有值=新公司），初盘是col0为空的行，不要混入
- 凯利指数在 `col8/9/10`，与赔率同一行
- 亚盘列：`col1=上盘水` `col2=盘口(字符串)` `col3=下盘水` `col4=时间`
- 大小球列：`col1=大球水` `col2=盘口(字符串)` `col3=小球水`
- 让球指数列：`col1=让球数` `col2=胜赔` `col3=平赔` `col4=负赔` `col5-7=概率` `col8=返还率` `col9-11=凯利`
- 过滤 `'最高值'/'最低值'/'平均值'/'离散值'`

### Step 2: 计算市场共识 + 逐公司EV

```
market_prob_w = (1/average_win_odds) / total_implied_prob
fair_odds_w = 1 / market_prob_w
company_ev_w = (company_odds_w / fair_odds_w - 1) × 100
```

| EV范围 | 含义 | 操作 |
|:--|:--|:--|
| > +10% | 强烈正EV | 高置信度，检查凯利>1.0 |
| +5% ~ +10% | 正EV | 中等置信度，可纳入 |
| 0% ~ +5% | 微正EV | 接近公平，谨慎 |
| < 0% | 负EV | 不推荐 |

### Step 3: 凯利三分类验证

```
凯利 > 1.0:  正EV确认（公司慷慨=赔率高于公平）
凯利 0.95~1.0: 中性
凯利 < 0.95:  负信号（公司控赔=该结果不易出）
```

### Step 4: 亚盘/大小球辅助信号

```
亚盘升盘 → 让球方受热 → 支持让球方
亚盘降盘 → 资金流出 → 支持下盘
上盘水<0.80 → 上盘大热
小球水<0.75 → 强烈小球预期
```

### Step 5: 生成6种玩法投注单

| 玩法 | 数据来源 | 说明 |
|:--|:--|:--|
| 胜平负 | 竞彩官方欧赔 | 选EV>0的选项 |
| 让球胜平负(-1) | 让球指数表(竞*官*行) | 用让球市场的概率分布算EV |
| 比分 | 模型预测 | 选2-3个比分，标注参考赔率 |
| 总进球 | 大小球盘口 | 选小球OR大球方向 |
| 半全场 | 模型推断 | 参考赔率以柜台为准 |
| 混合过关 | 组合前5种 | 只串正EV方向，2串1为主 |

**200元预算分配模板**（实战验证）：
```
胜平负 30元  |  让球胜平负 40元  |  混合过关 30元
比分 35元    |  总进球 35元      |  半全场 30元
```

### Step 6: 投注单格式

每张票 `🧾 票N` 标题 + 表格框。含过关方式、各场选项+赔率、金额、奖金、逻辑。详见 `scripts/odds_model_v7.py`。

## 已知局限

- 公司名脱敏后无法直接定位唯一公司，但不影响 EV 计算（只需区分不同公司）
- 凯利指数基于 Excel 内预计算值，不同数据源的计算基准可能不同
- 亚盘/大小球的"数值"与"字符串"混排（如 `0.5` 和 `半球` 交替出现），解析时需分别处理
- 让球指数（让球玩法）的列结构与欧赔亚盘不同，需单独解析
