# 体彩竞彩 vs 外盘规则对照（权威口径）

> 更新：2026-06-27  
> 用途：出方案、解析票据、对比赔率前**必须先读**。禁止把外盘术语/盘口直接写成竞彩可买玩法。

## 一、命名对照（同一概念，两套叫法）

| 竞彩官方/API | 口语/旧文档 | 说明 |
|:--|:--|:--|
| HAD | SPF / 胜平负 | 标准 1X2，90分钟含补时 |
| HHAD | RQSPF / 让球胜平负 | **整数让球** ±1/±2/±3，非亚盘小数 |
| CRS | 比分 | 31 项精确比分 |
| TTG | 总进球 | 0–7+ 共 8 档，**不是大小球 2.5** |
| HAFU | BQC / 半全场 | 半场结果×全场结果，共 9 项 |
| — | 上下单双 | 竞彩足球 API **无此玩法**，勿推荐 |

**外盘术语禁止直译成竞彩：**

| 外盘 | 竞彩对应 | 能否直接买 |
|:--|:--|:--|
| 亚盘 受0.5/让0.75 | 无等价；最接近 HHAD 整数让球 | ❌ 不能直接下单 |
| Over/Under 2.5 | TTG 总进球档位（0–7+） | ❌ 不是同一个玩法 |
| Draw No Bet | 无 | ❌ |
| 双方进球 Yes/No | 无 | ❌ |
| Cash Out / 兑现 | 竞彩足球一般无 | ❌ |

## 二、体彩独有规则（外盘没有）

### 2.1 场次开放制（单关 vs 仅过关）

**所有玩法**（HAD/HHAD/CRS/TTG/HAFU）均按场次开放，并非「胜平负永远可单关」。

核实 API（出方案前必做）：

```
GET https://webapi.sporttery.cn/gateway/uniform/football/getFixedBonusV1.qry?clientCode=3001&matchId=XXX
→ value.oddsHistory.singleList
```

| single 值 | 含义 |
|:--|:--|
| `{"single": 1, "poolCode": "HAD"}` | 该玩法**可单关** |
| `{"single": 0, "poolCode": "HAD"}` | 该玩法**仅过关**（须至少 2串1） |

**2026-06-27 世界杯实测示例：**

| 场次 | 可单关 | 仅过关 |
|:--|:--|:--|
| 072 约旦 vs 阿根廷 | HHAD、CRS、TTG、HAFU | HAD 未开售 |
| 068 巴拿马 vs 英格兰 | HHAD、CRS、TTG、HAFU | HAD 未开售 |
| 069 哥伦比亚 vs 葡萄牙 | HAD、CRS、TTG、HAFU | HHAD |
| 071 阿尔及利亚 vs 奥地利 | CRS、TTG、HAFU | **HAD、HHAD** |
| 070 刚果金 vs 乌兹别克 | CRS、TTG、HAFU | **HAD、HHAD** |

**两层陷阱（必须区分）：**

1. **未开售**：`oddsList` 中该 poolCode 的 h/d/a 为空 → 根本选不了该玩法  
2. **仅过关**：玩法在售，但 `single:0` → 不能单关，必须串至少一场

### 2.2 开售缺失（与单关无关）

部分场次**不售 HAD**，只售 HHAD（如 068 巴拿马 vs 英格兰）。  
此时口语「英格兰胜」在竞彩**无法直接表达**；可选 HHAD 选项含义完全不同（如巴拿马+2 胜 ≠ 英格兰胜）。

### 2.3 奖金机制

| 投注方式 | 奖金类型 | 说明 |
|:--|:--|:--|
| **任意玩法单关** | **浮动奖金** | 按该选项全网投注比例分配；截止后公布 SP，可能与票面赔率不同 |
| **任意玩法过关** | **固定奖金** | 出票时组合赔率锁定 |

> 常见错误：以为只有 CRS/TTG 单关才浮动——**所有单关均浮动**。

### 2.4 过关硬性限制

1. **同一场比赛的不同玩法不能混串**（如墨西哥 HAD × 墨西哥 TTG = 非法）  
2. **不同运动项目不能混串**  
3. **木桶原则**：关数上限 = 所含玩法最低关数（CRS/HAFU 最高 4 关，TTG 6 关，HAD/HHAD 8 关）  
4. **无设胆**、竞彩足球一般**无 Cash Out**  
5. **投注基数 2元/注**（每注基本金额，[官方规则](https://www.sporttery.cn/help/60333.html?gid=2)）；票面 = `注数 × 2元 × 倍数`；倍数条文 2–99，App 常见 1–50；单票最高 **20000 元**  
6. **出购买建议前必须量化**：Markowitz/Kelly 连续金额 → `quantize_stake` / `quantize_budget`（`scripts/kelly_engine.py`）。外盘可下任意金额，竞彩不可

### 2.5 让球计算（HHAD）

`goalLine` 正数 = 主队受让，负数 = 主队让球。仅整数让球。

```
主队受让2球 (+2)：
  胜：主队进球+2 > 客队进球
  平：主队进球+2 = 客队进球
  负：主队进球+2 < 客队进球
```

**HHAD 选项 ≠ HAD 选项**：HHAD「主胜」是让球后的主胜，不是原始比分主胜。

### 2.6 半全场（HAFU/BQC）不受让球影响

中奖判定用**原始比分**的半场胜平负 × 全场胜平负，**不做让球调整**。  
（HHAD 让球与 HAFU 半全场是两套独立规则。）

## 三、外盘规则（分析用，不可当竞彩出票依据）

| 项目 | 外盘常见做法 |
|:--|:--|
| 盘口 | 亚盘小数（0/0.5/0.75/1…）、欧盘 1X2 |
| 大小球 | 2.5、2.75、3 等连续盘口 |
| 下单 | 通常单场任意玩法可下 |
| 奖金 | 赔率锁定（水位变动除外） |
| 用途 | **模型参考、去水概率、定价偏差** |

**正确用法：** 用外盘（Pinnacle/Bet365 等）估计真实概率 → 与竞彩赔率比 EV → 输出时必须**映射回竞彩可买玩法**并核实 `singleList`。

## 四、出方案强制流程

```
1. getMatchListV1 → 确认每场在售 poolCode（HAD 是否缺失）
2. getFixedBonusV1 → singleList 确认单关/仅过关
3. 若用户要的方向无对应玩法 → 明确写「竞彩不可买/需改用 HHAD xxx」
4. 若 single:0 → 写「须 2串1+」，给出合规替代结构
5. 单关标注「浮动奖金」；串关标注「固定奖金」
6. 禁止把外盘亚盘/大小球写成竞彩投注项
```

## 五、常见错误对照表

| 错误表述 | 正确表述 |
|:--|:--|
| 「CRS/BQC 不能单关」 | 按场次开放；世界杯多数场次 CRS 可单关，须查 API |
| 「胜平负永远可单关」 | HAD 也按场次开放；071/070 等场次 HAD 仅过关 |
| 「英格兰胜」当 068 出票选项 | 068 无 HAD；只能 HHAD 具体选项（含义不同） |
| 「买大小球 2.5 小球」 | 竞彩用 TTG「0球/1球/2球…」档位 |
| 「客队受让0.5」 | 外盘术语；竞彩用 HHAD 整数让球 closest match |
| 「BQC 结算要让球调整」 | **错误**；BQC/HAFU 用原始半场+全场结果 |
| 「单关固定赔率」 | 单关为浮动 SP；过关才固定 |
