# 投注回测计算器

脚本路径：`.\scripts\wc2026_backtest.py`

## 用法

```bash
# 命令行传入比分
python3 wc2026_backtest.py 墨西哥 2 1 南非 韩国 1 1 捷克

# 不传参数则使用示例比分
python3 wc2026_backtest.py
```

## 支持的玩法

| 玩法 | 计算函数 | 说明 |
|:|:|:|
| SPF | `calc_spf(hg, ag)` | 胜平负 |
| RQSPF | `calc_rqspf(hg, ag, handicap)` | 让球胜平负（handicap负数=主队让球） |
| CRS | `calc_crs(hg, ag)` | 比分（含"胜其它""平其它""负其它"） |
| TTG | `calc_ttg(hg, ag)` | 总进球（0-6, 7+） |
| BQC | `calc_bqc(hg_ht, ag_ht, hg, ag)` | 半全场（上半场+全场结果） |

## 关键 Pitfall

1. **BQC选项映射**：代码内部用中文（"胜胜"），投注方案用英文（"HH"）。匹配时必须映射：
   ```python
   bqc_map = {'HH':'胜胜','HD':'胜平','HA':'胜负','DH':'平胜','DD':'平平','DA':'平负','AH':'负胜','AD':'负平','AA':'负负'}
   ```

2. **让球数方向**：`handicap=-1` 表示主队让1球（主队实力强），`handicap=+1` 表示客队让1球

3. **CRS比分边界**：主队进6球以上或客队进3球以上 → "胜其它"；4:4以上平局 → "平其它"

4. **容错2串3计算**：= 2个单关 + 1个2串1，共3注。不是简单的组合数。

5. **BQC让球陷阱**：BQC赛果也受让球影响。韩国2-1捷克（让-1球），半场0-0让1球后=0-1（负），所以BQC"胜胜"不中。投注BQC时必须用让球后的半场+全场结果来判断中奖条件，而非原始比分。
   ```
   原始半场0-0 → 让1球后半场0-1 = 负 
   原始全场2-1 → 让1球后全场1-0 = 胜
   因此BQC=RQ"负胜"(AH)，非"胜胜"(HH)
   ```
